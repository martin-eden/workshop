Here is PNM codec. It's two functions: one for load, other for save.

Load requires data stream, returns image data and settings table.
Save requires image data and settings table, writes to output stream. [1]

[Image] is our internal image class.

Conceptually it protects image data, access should be only via
GetPixel(), SetPixel().

Class requires Settings{} table to be configured before access.

[1]: Stream, not string

  Stream is the base. String is actually stream when you're implementing it.

  DS3231 codec uses string (unlike Itness codec).

  That's because DS3231 data is fixed-length structure (and just 12 bytes).

  Here we're dealing with variable-length context-based data. Stream.
