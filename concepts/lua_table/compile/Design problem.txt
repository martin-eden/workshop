Before explaining problem I'll explain existing architecture

  Architecture

  That's graph serializer. It receives Lua table with possible
  cross-references. In short, it receives graph.

  First, it traverses this table to create execution plan for
  serialization. "Execution plan" is commonly called "AST"
  (annotated syntax tree) in this field.

  Execution plan is tree (not graph!) of records to process.

  Originally code here created AST only for trees (not graphs!).
  When later we wrote code for serializing graphs it wanted to
  reuse this code for trees. It provides us "tables to ignore" map.

  "Tables to ignore" map argument currently named "NamedNodes_Map".
  If while traversing tree we meet subnode from this map,
  we don't dive. We emit record

    { type = 'name', value = NamedNodes_Map[Data] }

  And later graph serializer receives our AST and it's his problem
  how to deal with that named nodes.


  Tree serialization

  Okay, we have execution plan (AST) which is tree of typed records.

  We have generic function to traverse tree of typed records
  and execute handlers for them. It's called [struc.compile].

  So now we need to create handlers for typed nodes that we know.
  And then we call [struc.compile].

  Handler receives record and just writes string somewhere.

  Small problem (which is handled) that it sometimes need partial state.

    For example if we serializing "['a'] = 1" we write "[" and then call
    handler to serialize string "a". String "a" can be serialized with
    long quotes: "[[a]]". And if string serialized this way we will
    write "[[[a]]]" which will parse to ("[a" "]") and that sucks.

    So our handlers are in one file to have common state as upvalue
    (or else we must provide that common state as argument).
    After each write we update "<last_char>" to last character wrote.

    That's specially for string serializer. If serialized string
    starts with "[" and <last_char> is "[", we just emit " " before
    emitting quoted string. To break syntax clash.


  Graph serialization

  Graph serializer is here again and it wants to use our handlers.
  Also it has it's own handlers for that additional AST nodes.

  Problem that is unsolved that graph serializer node handlers
  can emit string ending with "[".

  All serialization nodes is called from [struc.compile].
  Which received graph AST and node handlers map.
  And node handlers map is tree node handlers and graph node handlers.

  Tree handlers know last character they wrote. Graph handler's
  can emit string ending with "[". Tree handler won't know about this,
  <last_char> is internal closure for him and his friends in that file.
