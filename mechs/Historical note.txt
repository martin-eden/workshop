What is here and why?

"mechs/" dir originally was for computational-heavy smart stuff.

Years later "concepts/" directory was created. Originally it was
renamed from "formats/" directory with file format codecs.

Later I wanted to repurpose "concepts" for data-centric structures.
But they were growing with utility functions, becoming smarter
and computational-heavy...

Truth is that there is no natural distinction between data and code.
Between data, data structures and algorithms.

So this directory is historical artifact still containing code
used by older projects and internal parts.

I don't want to move it to "concepts" without analysis and refactoring.
And (as usually) I perceive that writing new things or redesigning
another parts is more important for progress.

Still, it's not abandoned code. It's not rotten. It should work.

Problem with it is experimental interfaces and lack of documentation.

So generally nothing is added here anymore. Some parts can migrate
after rework. Some parts may be substituted and removed.

-- Martin, 2026-06-12
